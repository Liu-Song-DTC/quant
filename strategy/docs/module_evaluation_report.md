## 四、组合层评估

### 4.1 代码分析 (portfolio.py)

**仓位计算公式**:
```python
base_position = score * (1.0 / risk_vol)
```

**问题**:
1. `score`被限制在[0,1]范围，但当前因子值可能超出
2. 低波动股票获得更高仓位，但验证时未用波动率调整收益

**回撤控制**:
```python
if drawdown > 0.05:  max_gross_exposure = 0.80
if drawdown > 0.10:  max_gross_exposure = 0.50
if drawdown > 0.15:  max_gross_exposure = 0.30
```

**问题**: A股5%回撤是常态，可能频繁降仓

### 4.2 当前表现（基于信号评估）
| 指标 | 值 | 评价 |
|------|-----|------|
| Top-10平均收益 | 4.78% | 良好 |
| Top-10夏普 | 0.97 | 良好 |
| 五分位单调性 | Q5-Q1=2.37% | ✓ 通过 |

### 4.3 优化建议
1. **放宽回撤阈值**: 5%→10%, 10%→15%
2. **仓位公式调整**: 不要用score的绝对值，用排名百分位
3. **行业权重**: 对弱势行业降低配置

## 五、执行层评估

### 5.1 代码分析 (bt_execution.py)

**交易成本**:
- 佣金: 0.15%
- 滑点: 0.15%
- 单边总成本: 0.30%
- 年化（12次调仓×50%换手）: ~3.6%

**A股限制**:
- 100股整手限制 ✓ 已处理
- T+1 ✗ 未处理（用当天收盘价可能有前视）

### 5.2 问题诊断
1. **前视偏差**: 用当天收盘价生成信号并交易
2. **交易成本**: 年化3.6%会吃掉较多收益

### 5.3 优化建议
1. **修复前视**: 用前一天收盘价生成信号，或用开盘价执行
2. **降低换手**: 考虑信号平滑或调仓门槛
