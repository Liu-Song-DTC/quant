## 三、Phase 1 详细设计

### Exp A1: BP7扩展

**现状**: BP7质量最高(WR=55.6%, MR=1.82%)但仅2.8%信号
**根因**: 需读懂BP7的触发条件，分析拒绝路径
**改动**:
```
1. 分析BP7买入vs未买入的signal_level/trend_type分布
2. 找到限制BP7信号量的条件
3. 放宽筛选条件（如扩大价格区间、放宽量能要求）
4. 目标: BP7占比 2.8% → 5-8%
```
**验证**: 回测2021-2026，对比BP7信号占比和总体Sharpe
**文件**: `chan_theory.py` BP7段 + `signal_engine.py` 筛选逻辑

### Exp A2: B2放宽

**现状**: B2仅1.0%信号，条件过严
**条件链**: B2需: ①已发生B1 → ②次级别回调 → ③不破B1低点 → ④MACD确认
**改动**:
```
1. 扩大B2检测窗口(当前只在B1后50bar内检测→100bar)
2. 放宽回调深度要求(当前需回调到0.382-0.618黄金分割)
3. 降低MACD确认阈值
4. 目标: B2占比 1.0% → 3-5%
```
**验证**: 回测2021-2026，B2信号量和WR
**文件**: `chan_theory.py` detect_second_buy_point

### Exp A3: B5分析+门控

**现状**: B5(趋势启动)占4.3%，无质量分析
**动作**:
```
1. 跑分析: B5的signal_level/trend_type/mom vs future_ret
2. 建立B5门控(类似B4的SL>=2)
3. 如果SL=0也亏损→过滤
```
**验证**: B5买点WR提升

### Exp A4: B4收严（备选）

**现状**: B4用5个硬AND条件，无纵向分层（除了SL>=2门控）
**可选改动**: 缩量要求从1.1x→0.9x，或趋势强度从MA20>MA60→MA20>MA60*1.02
**风险**: 信号减少可能降低收益，需谨慎

## 四、Phase 2 详细设计

### Exp B1: 市场状态自适应仓位

**现状**: 全阶段满仓(max_positions=5)，熊市照样重仓
**改动**:
```yaml
# factor_config.yaml
market_adaptive:
  enabled: true
  regime_exposure:
    strong_bull: {max_positions: 8, exposure: 1.0}
    bull: {max_positions: 6, exposure: 0.9}
    neutral: {max_positions: 4, exposure: 0.6}
    bear: {max_positions: 2, exposure: 0.3}
```
**市场状态判断**: 用已有的market_regime_detector输出(指数MA20/MA60/MA120排列)
**验证**: 2021-2023熊市回撤是否减小，2025牛市收益是否保持

### Exp B2: 行业分散约束

**现状**: 选股无视行业集中度，可能5只全是同一板块
**改动**:
```
portfolio.py: 选股时增加行业分散约束
- 同行业最多2只
- 同概念板块最多1只（如果concept_heat>0.5）
```
**验证**: 行业集中度下降，最大回撤减小
